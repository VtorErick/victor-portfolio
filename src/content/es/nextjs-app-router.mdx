# El Cambio de Paradigma

Next.js 13 introdujo el App Router, y en la versión 15 (usada en este sitio) ya es el estándar indiscutible.

## Server Components por Defecto

En el directorio `app/`, todo es un Server Component a menos que digas lo contrario. Esto significa:
- **0kb de JS** enviado al cliente para componentes estáticos (como mi Footer o Textos).
- Acceso directo a base de datos o sistema de archivos (fs) sin exponer secretos.

## Layouts Anidados

El archivo `layout.tsx` me permite definir una estructura persistente (como el NavBar) que no se desmonta al navegar entre páginas, preservando el estado del scroll y animaciones.

```tsx
// src/app/layout.tsx
export default function RootLayout({ children }) {
  return (
    <html lang="es">
      <body>
        <LanguageProvider>
           {children}
        </LanguageProvider>
      </body>
    </html>
  )
}
```

Esta arquitectura reduce drásticamente el First Contentful Paint (FCP).
